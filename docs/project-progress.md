# 项目进度说明

最后更新：2026-07-22

## 1. 项目定位

本项目是面向盲人跑步辅助腰带的 Android App 与 ESP32 固件。当前目标是先完成可靠的路线记录、路线选择、运动过程显示和腰带硬件调试，再逐步接入正式的方向决策与自动振动反馈。

当前已经形成以下基础闭环：

    记录路线 -> 本地保存 -> 选择路线 -> 开始运动 -> 显示路线进度
                                          |
                                          -> Debug BLE 手动测试与正北自动方向测试

“定位与路线匹配 -> 切线方向 -> 人体方向 -> 稳定电机选择 -> BLE 脉冲振动”的正式运动链路已经接通。当前仍需完成真实路线闭环测试和震动参数标定。

## 2. 技术栈

- Android / Kotlin
- Jetpack Compose + Material 3
- ViewModel + StateFlow
- 高德 2D 地图 SDK
- 高德定位 SDK
- App 私有目录 JSON 存储
- Android BLE GATT Client
- ESP-IDF 6.x + NimBLE GATT Server
- JUnit 4

应用最低支持 Android 8.0（API 26），当前 compileSdk 和 targetSdk 为 35。

## 3. 当前页面

App 底部包含“记录、运动、已保存”三个入口。

Debug 构建中，在2.5秒内连续点击“运动”5次可进入隐藏测试工具。Release 构建不会提供虚拟 GPS 和电机调试功能。

## 4. 已完成功能

### 4.1 记录页面

- 使用高德 2D 地图作为主界面。
- 记录地图固定为20级缩放，并禁用双指、双击等缩放手势。
- 申请精确定位权限并显示当前位置蓝点。
- 进入记录页面后，首次获得有效定位会自动将地图居中到当前位置。
- 高精度定位，定位请求间隔为3秒。
- 点击 GO 开始记录。
- 开始后进行15秒 GPS 预热，预热期定位点不保存。
- accuracy 为空或精度大于8米的点会被丢弃。
- 地图实时绘制已接受定位点形成的路线。
- 折叠底栏显示已保存点数和路程。
- 上拉底栏显示时间、平均速度、当前精度、定位状态和丢弃点数。
- 支持暂停、继续和长按1.5秒结束记录。
- 暂停期间停止采点并冻结计时，继续后保留已有数据。
- 只有记录结束后才询问是否保存。
- 选择保存后输入路线名称；选择放弃后清空本次记录。

当前 RouteRecorder 对外能力：

    startRecord / pauseRecord / resumeRecord / stopRecord
    getPointCount / getCurrentPoints
    getLatestAccuracy / getLatestReceivedAccuracy
    isLatestPointAccepted / getDiscardedPointCount
    getWarmupRemainingSeconds
    saveRoute / loadRoutes / deleteRoute

### 4.2 路线存储与已保存页面

- 路线保存为 App 私有目录 filesDir/routes/ 下的 JSON 文件。
- App 重启后仍可读取路线。
- 已保存页面支持列表、刷新和空状态。
- 点击路线进入详情。
- 长按路线后经过二次确认删除。
- 删除失败时显示错误提示。
- 路线详情显示名称、ID、创建时间、点数和平均定位精度。
- 详情地图显示起点、终点、路线和末端切线箭头。
- 路线详情不显示逐点明细列表。

数据模型保持精简：

    RoutePoint: latitude, longitude, timestamp, accuracy
    RouteRecord: id, name, createdAt, points

### 4.3 路线切线与进度算法

RouteTangentCalculator 可以根据路线和当前位置计算：

- 最近路线线段
- 在线段上的投影比例
- 到路线的距离
- 路线切线方向角
- 沿路线累计距离
- 路线总距离
- 0.0 到 1.0 的路线进度

路线详情使用末端切线；运动页面使用当前位置切线和路线进度。

NavigationVibrationPlanner 已实现导航震动的纯决策逻辑：

- 用路线切线方向减去当前朝向，得到相对人体方向。
- 将360度划分为八个45度扇区。
- 电机编号沿顺时针排列：1前、2右前、3右、4右后、5后、6左后、7左、8左前。
- 输出当前状态、建议电机、目标方向、相对方向、偏离距离和剩余距离。
- 定位精度差、偏离路线、缺少朝向或到达终点时不输出电机编号。
- 偏离路线阈值为 max(30米, 定位精度的2倍)。
- 到达条件为路线进度不低于95%，且剩余距离不超过10米。

该决策已经接入 SportViewModel。开始和继续运动时恢复计算；暂停、结束、定位失败或定位权限撤销时停止给出有效电机建议。运动页使用手机背面朝向作为人体方向，并根据实时位置将磁北方位修正为真北方位。

### 4.4 运动页面

运动页面已不再是占位页面，目前实现：

- 自动显示当前位置。
- 进入运动页面后，首次获得有效定位会自动将地图居中到当前位置。
- 已经获得当前位置时，选择路线不会再把地图相机强制移动到路线起点。
- GPS 质量、精度数值和三格信号状态。
- 从本地已保存路线中选择运动路线。
- 路线列表包含加载、错误、空状态和刷新。
- 没有路线时可以前往记录页面。
- 只有“路线有效 + 精确定位权限 + 已获得位置”时才能开始。
- 地图显示路线、起点、终点和当前位置。
- 运动地图固定为20级缩放，并禁用双指、双击等缩放手势。
- 用户手动移动地图后暂停自动跟随，可点击定位按钮恢复。
- 定位更新不会反复重置用户缩放。
- 开始运动后显示总距离、总时长、实时配速和路线完成度。
- 已完成路段使用不同颜色绘制。
- Bottom Sheet 支持折叠和上拉展开。
- 支持暂停、继续和长按1.5秒结束。
- 继续后恢复计时、距离和路线进度。
- 运动结束后只显示运动结果并返回运动首页，不保存运动记录。
- 准备页显示腰带状态，可以点击扫描并连接。
- 开始运动时若腰带尚未连接，会自动请求 BLE 权限并扫描 BeltMotor。
- 使用旋转矢量传感器持续获取手机背面朝向。
- 使用当前位置和时间计算磁偏角，将磁北朝向转换为路线切线使用的真北朝向。
- 根据导航决策自动发送1至8号电机命令，并避免重复发送相同目标。
- 腰带状态显示当前自动选择的电机编号。
- 暂停、结束、偏离路线、定位或朝向失效时发送停止命令。
- 连接失败后可以点击腰带状态重新扫描。

运动指标过滤规则：

- 精度必须在0至25米。
- 小于2米的移动不累计。
- 超过12m/s的跳点不累计。
- 相邻点时间间隔超过30秒不累计该段。
- 实时配速使用最近60秒的有效移动段计算。

路线进度仅接受精度不超过25米、且距离路线在合理范围内的定位点，并阻止进度倒退。

### 4.5 Debug 虚拟 GPS

Debug 页面提供三种固定场景：

- 直线路线：持续向东，精度3米。
- 直角转弯路线：先向北再向东。
- 精度过滤路线：交替发送3米和15米精度点。

虚拟点每3秒发送一次，同时驱动记录数据源和地图当前位置。Release 中的同名 Provider 始终不可用。

当前虚拟定位由手机内部生成，尚不支持电脑通过 ADB、HTTP 或 Socket 逐点发送自定义坐标。

### 4.6 Debug BLE 与 ESP32 固件

Debug App 可以：

- 请求 Android 12 及以上的 BLE 权限。
- 扫描并连接设备名 BeltMotor。
- 使用八方向轮盘选择1至8号电机。
- 点击轮盘中心 STOP 停止全部电机。
- 断开或退出 Debug 页面时停止电机。
- 使用旋转矢量传感器计算手机背面朝向。
- 支持固定正北路线的自动电机测试。
- 自动测试期间禁用手动轮盘，避免两种控制方式冲突。
- 朝向无效、传感器不可靠、BLE 断开或停止测试时停止电机。

正北路线测试采用以下安装约定：

    手机竖直放在人体正前方
    屏幕朝向身体，背面朝外
    手机背面方向代表人体正前方

预期结果：

    面北 -> 1号电机
    面东 -> 7号电机
    面南 -> 5号电机
    面西 -> 3号电机

BLE 协议：

    Service UUID: 8f8a0001-8f4b-4f5b-9f2b-5e7a1f000001
    Write Characteristic UUID: 8f8a0002-8f4b-4f5b-9f2b-5e7a1f000001
    ASCII 1..8 -> 启动对应电机
    ASCII 0    -> 停止全部电机

belt_motor_test 固件已经实现：

- 八路1kHz、8-bit PWM。
- 同时只启动一个电机，当前占空比为128。
- BLE 与 UART 使用相同命令。
- BLE 客户端断开时停止全部电机。
- Windows ASCII 临时路径编译、烧录和串口监视脚本。

固件 GPIO 映射：

    1 -> GPIO4
    2 -> GPIO13
    3 -> GPIO14
    4 -> GPIO18
    5 -> GPIO19
    6 -> GPIO21
    7 -> GPIO22
    8 -> GPIO23

### 4.13 真实运动导航震动

- 正常引导改为方向电机短振 320 ms、停振 880 ms 的重复脉冲，不再持续通电。
- 电机方向加入 8 度扇区滞回，降低跑动晃动导致的相邻电机反复切换。
- 偏离路线时使用左侧 7 号与右侧 3 号电机交替双振，区别于方向引导。
- 到达终点时使用正前方 1 号电机连续三次短振，完成后自动停止。
- 定位精度不足、朝向不可用、路线无法匹配、暂停和结束运动时不输出导航震动。
- 运动中 BLE 断连或连接异常后等待 3 秒自动重新扫描连接。
- 运动路线卡实时显示方向引导、偏航、到达、定位不足和朝向等待状态。
- ESP32 协议保持 ASCII 0 至 8 不变，无需重新修改固件。

## 7. 已确认的产品决定

- 只有记录页面保存路线。
- 运动结束不保存，也不弹出保存窗口。
- 心率和热量暂不考虑，相关状态字段不作为当前验收项。
- Debug 虚拟 GPS 和手动电机控制不进入 Release 产品功能。
- 手机安装约定为竖直放在人体正前方、屏幕朝向身体、背面朝外。
- 当前路线文件格式暂不加入导航状态、电机状态等派生字段。

## 8. 建议下一步

建议按以下顺序继续：

1. 使用短距离真实路线完成低强度、可立即停止的运动闭环测试。
2. 核对地图切线、磁偏角修正和实际电机方向是否一致。
3. 在真机上标定方向脉冲、偏航双振、到达三振和 8 度滞回参数。
4. 增加 BLE 写入失败的运动页错误提示与导航降级策略。
5. 根据测试反馈决定是否增加“重新回到路线”的独立确认节奏。
